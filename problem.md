# 当前代码问题与工作进度

## 当前状态
当前项目已经从“能跑起来的三维装箱 baseline”往“更可信的实验工具”推进了一步，但还没有达到数学建模比赛里可直接支撑论文级实验的程度。

截至 2026-04-20，已经完成的核心改进有：
- 缓存已绑定 `cacheVersion` 和代码指纹，算法实现变化后旧缓存会失效。
- 已加入统一的解合法性校验，并在缓存恢复后再次校验。
- 非法结果不会参与可视化，也不会写入缓存。
- 报告里已经展示合法性、校验错误、随机种子等关键信息。
- 结果目录中可额外导出最佳合法解的文本明细，便于复盘。

当前更准确的判断是：项目已经具备“基础可信度保障”，但“严格可复现实验”和“研究型批量分析输出”仍未完成。

## 问题清单与进度

### 1. 缓存没有绑定代码版本 [已完成]
这个问题已经修复。

当前 `testRunner.py` 会为求解相关代码生成 `codeFingerprint`，并与 `cacheVersion` 一起写入缓存。读取缓存时会校验：
- `cacheVersion` 是否一致
- `codeFingerprint` 是否一致
- 缓存中的解在当前代码下是否仍然通过合法性校验

因此，算法代码修改后继续命中旧缓存、导致静默复用过期结果的问题，已经被压住。

### 2. 缺少严格的解合法性校验 [基础版已完成]
这个问题已经完成基础修复。

新增的 `solutionValidator.py` 已经统一校验：
- 货物是否越界
- 货物之间是否重叠
- 是否存在重复放置
- 旋转编号是否合法
- 总重量是否超载
- `totalWeight`、`totalVolume`、`volumeRate`、`weightRate` 与实际放置结果是否一致

同时，缓存恢复后的解也会重新校验；非法结果会：
- 在日志中输出校验错误
- 被标记为 `INVALID`
- 跳过缓存写入
- 跳过可视化

需要注意的是，这里完成的是“几何合法性 + 汇总指标一致性”校验，不包含稳定性、支撑面、重心等更高阶业务约束。

### 3. 随机实验默认不可复现 [进行中]
这个问题还没有真正解决，只是补上了一部分可追踪信息。

当前进展：
- 报告中会记录每次运行使用的 `seed`
- 解文本中也会记录 `seed` 和 seed 模式
- 配置层本身支持 `useTimeSeed: false`，可以切到固定种子

仍然存在的问题：
- 默认配置仍然使用时间种子
- 每组参数默认仍只跑一次
- 还没有“多次重复试验 + 统计均值/方差/最优值”的机制

所以现在更像是“可追踪”，还不是“默认可复现”。

### 4. 搜索能力偏弱 [未开始]
这一项目前没有实质改进。

当前优化过程仍然主要依赖：
- 打乱物品顺序
- 用同一个 `simplePacking` 风格的解码逻辑生成解

这意味着它仍然偏向基础启发式框架，搜索空间探索能力有限，解质量上限容易卡住。

### 5. 约束建模能力不足 [未开始]
这一项仍然基本未动。

当前结构依然主要围绕尺寸、重量和放置坐标展开，尚未系统支持：
- 稳定性与重心
- 易碎品限制
- 堆叠规则
- 装载顺序
- 多容器/多车型分配

虽然 `Item` 增加了 `typeId` 字段，便于结果导出和追踪原始类型，但这还不构成约束建模能力升级。

### 6. 报告更偏演示，缺少研究输出 [部分完成]
这一项有进展，但还不够。

已经完成的部分：
- 报告加入了合法性字段
- 报告会列出非法解的校验错误
- 可以导出最佳合法解的结构化文本
- 可视化阶段只处理合法结果

仍然缺少的部分：
- 面向批量分析的 CSV/JSON 汇总表
- 重复试验统计
- 跨测试用例的基线对比
- 便于论文整理的统一数据接口

因此，这一项目前应视为“从演示输出向研究输出迈出了一步”，但还没有完成。

## 下一步建议
如果目标是尽快把项目推进到“比赛可用的稳定实验工具”，下一阶段建议按下面顺序推进：

1. 先补可复现实验机制
   将默认配置改为固定种子，增加重复试验次数和统计汇总。
2. 再补机器可读结果导出
   输出 CSV/JSON 汇总表，方便后处理、画图和论文整理。
3. 然后增强搜索能力
   引入更强的邻域设计、候选解扰动和多策略比较。
4. 最后扩展业务约束
   根据题目需要引入稳定性、堆叠和多容器等现实约束。

## 总体判断
如果目标只是做一个“能跑的三维装箱程序”，当前项目已经超过 baseline 阶段。

如果目标是作为数学建模比赛中的核心实验平台，当前状态更接近：
- 已完成：结果可信度基础建设
- 部分完成：结果记录与导出能力
- 尚未完成：默认可复现实验、强搜索能力、现实约束建模

也就是说，最危险的“缓存污染”和“非法解混入结果”问题已经显著缓解，但真正决定比赛上限的实验设计能力和建模能力还在后面。
