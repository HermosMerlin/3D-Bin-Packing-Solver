# 3D Bin Packing Solver (三维装箱算法)

## Project Overview
This project is a 3D Bin Packing Solver implemented in Python. It uses a space-splitting heuristic algorithm to pack rectangular items into a single container, aiming to maximize volume utilization and weight capacity.

### Key Features:
- **Core Heuristic:** Implements a space-splitting strategy in `packingLogic.py`.
- **Optimization:** Uses iterative sequence permutation (Genetic Algorithm-inspired) in `optimization.py` to find better packing orders.
- **Visualization:** Generates high-quality 3D interactive visualizations using `Plotly`. Supports both HTML and PNG (via `kaleido`) exports.
- **Batch Testing:** Robust batch testing framework in `batchTest.py` that supports parameter grid searches and multiple test cases.

## Core Architecture
- `dataStructures.py`: Defines the foundational entities: `Item`, `Container`, `Space`, and `PackingSolution`.
- `packingLogic.py`: Contains the `simplePacking` function, which is the primary heuristic for placing items into available spaces.
- `optimization.py`: Contains `optimizePacking`, which wraps the core heuristic with an optimization loop to explore different item sequences.
- `visualizer.py`: Manages the generation of 3D plots, providing a clear visual representation of the packing results.
- `configManager.py` & `testCaseManager.py`: Handle project configuration and test case loading from JSON files.

## Getting Started

### Prerequisites
- Python 3.x
- Dependencies:
  ```bash
  pip install plotly numpy kaleido
  ```

### Running the Solver
To run the batch testing suite:
```bash
python batchTest.py
```

### Configuration
- **Global Settings:** Modify `config.json` to change default algorithm parameters (iterations, random rate) and output settings.
- **Test Cases:** Add or modify JSON files in the `test/` directory. Each file defines container dimensions, weight limits, and item types/counts.

## Development Conventions
- **Data Integrity:** Always use the `PackingSolution` class to track placed items and calculate utilization rates.
- **Surgical Updates:** When modifying the packing logic, ensure that the `Space` splitting logic remains consistent with the 3D coordinate system.
- **Visualization:** New features affecting item placement should be verified using the interactive HTML output generated by `Visualizer`.
- **Naming:** Follow standard Python `camelCase` for methods and variables as established in the current codebase.

## Directory Structure
- `test/`: Input test cases (JSON).
- `results/`: Output directory for logs, metrics, and visualizations (organized by timestamp).
- `__pycache__/`: Python compiled files.
